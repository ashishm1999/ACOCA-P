# ACOCA-P — Datasets

## `roadwork_dataset.jsonl`

Simulated capture from four Nerian SceneScan 3D depth cameras deployed at four
Melbourne roadwork sites (Hawthorn, Burwood, Malvern, and the Swanston/Collins
CBD control site). Records represent aggregated frame-level captures at roughly
one sample per camera per second across a 24-hour window.

- **Records**: 280,000
- **Format**: JSON Lines (one JSON object per line)
- **Size**: ~515 MB
- **Fields per record**: 15+ top-level keys including `site`, `sensor`,
  `depth_frame`, `detections`, `context`, and `quality`.
- **Context vocabulary**: uses the CoaaS ontology (`coaas:`) alongside
  `schema.org` and `sosa:` — matches the JSON-LD descriptors served by real
  CoaaS providers.

## Regenerating

```bash
python3 generate_dataset.py --n 280000 --out roadwork_dataset.jsonl --seed 20260703
```

The generator is deterministic under a fixed `--seed`.

## Field reference

Each record has the following top-level shape:

```jsonc

  "@id": "nerian:frame:nerian-h1:000000042",

  "@context": { "coaas": "http://schema.coaas.org/", ... },
  "seq": 42,
  "captured_at": 1782432042000,
  "captured_at_iso": "2026-07-01T00:00:42Z",
  "site":   { "site_id": "hawthorn-glenferrie-rd", "camera_id": "nerian-h1", ... },
  "sensor": { "make": "Nerian", "model": "SceneScan Pro", "firmware": "8.6.2", ... },
  "depth_frame": {
    "resolution": "1600x1200",
    "fps": 25,

```

## Loading into MongoDB

```bash
mongoimport --uri "$MONGO_URI" \
    --collection roadwork \
    --file roadwork_dataset.jsonl
```

## Provider inventory (thesis §9.2.3)

This dataset covers the roadwork provider family (~85,600 items in the thesis
counting scheme). The remaining five provider families (vehicles, incidents,
weather, speed zones, signals) are generated by the IoT Data Simulator at
runtime from the same seed distribution.
